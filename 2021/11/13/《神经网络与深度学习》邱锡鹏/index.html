《神经网络与深度学习》邱锡鹏

2021-11-13 读书笔记 DL, ML 评论 字数统计: 8k(字) 阅读时长: 29(分)

1 绪论

1.1 人工智能

人工智能（Artificial Intelligence，AI）：让机器拥有人的智能（感知、学习、认知）。

  • 感知：（计算机视觉、语音信息处理）
  • 学习：（监督学习、无监督学习、强化学习）
  • 认知：（知识表示、自然语言理解、推理、规划、决策）

1.2 机器学习

机器学习（Machine Learning，ML）：从数据中学习出一般规律，利用规律进行新的预测。

实际任务中使用机器学习模型的几个步骤：

  1. 数据预处理：去掉一些缺失特征的样本、去掉一些冗余的数据特征、对数值特征进行缩放和归一化
  2. 特征提取：提取高质量特征。如文本分类中去除停用词
  3. 特征转换：对特征降维（特征抽取和特征选择）和升维。常用方法：主成分分析（Principal Components Analysis，PCA）、线性判别分析（Linear Discriminant Analysis，LDA）等
  4. 预测：学习一个函数并进行预测

很多机器学习问题往往变成了**特征工程（Feature Engineering）**问题

1.3 表示学习

机器学习中表示特征的两种方式：

  • 局部表示 / 离散表示 / 符号表示：one-hot向量形式
  • 分布式表示：低维稠密向量 。

嵌入（Embedding）：我们使用神经网络将高维的局部表示空间 R∣D∣\R^{|D|}R∣D∣ 映射到一个非常低维的分布式表示空间RDR^DRD，D≪∣V∣D\ll|V|D≪∣V∣。机器学习中此过程称为嵌入。自然语言中词的分布式表示，经常叫做词嵌入。

表示学习（Representation Learning）：可以自动学习有效特征，提高机器学习模型性能的学习方法。

表示学习的关键：构建一定深度的多层次特征表示。因为要学习到一个好的高层语义表示（一般为分布式表示），通常要从底层特征开始经过多次非线性转换才能得到，因此选择深层结构。深层结构可以增加特征的重用性，从而指数级地增加表示能力。

传统机器学习的特征学习方法：主成分分析、线性判别分析、独立成分分析。

1.4 深度学习

深度学习（Deep Learning，DL）：从数据中学出一个深度模型。从数据中自动学习有效的特征表示。

深度学习早期主要用来进行表示学习，后来越多越多地用于处理更复杂的推理、决策问题。

深度学习所采用的的模型是神经网络模型（当然还有其他的模型），主要原因是神经网络模型可以使用误差反向传播算法，可以比较好地解决贡献度分配问题（Credit Assignment Problem，CAP）——一个系统中不同的组件或其参数对最终系统输出结果的贡献或影响。

1.5 神经网络

神经网络，是人工神经网络的简称，即一种模仿人脑神经网络的数学模型，由节点（人工神经元）互相连接组成。不同节点间的连接被赋予不同的权重，每个权重代表一个节点对另一个节点的影响大小。每个节点都代表一种特定的函数，由其他节点传来的信息经过相应权重进行计算，再输入到一个激活函数中得到一个新的节点值。

1.7 常用的深度学习框架

为什么需要框架：深度学习中，一般用误差反向传播算法进行参数学习，手工求梯度再写代码的方式非常低效。此外，深度学习模型的计算资源很多，需要在CPU和GPU之间无缝切换。因此，我们需要一种可以自动求梯度，且可以无缝在CPU和GPU之间切换的功能，于是产生了支持这些功能的深度学习框架。代表性框架：Theano、Caffe、TensorFlow、 PyTorch、飞桨（PaddlePaddle）、Chainer和MXNet等。

  • Theano：蒙特利尔大学。停止维护
  • Caffe：加州伯克利分校。针对卷积神经网络的计算框架，主要用于计算机视觉。（Caffe2已并入PyTorch）
  • TensorFlow：Google 。
  • PyTorch：Facebook、NVIDIA、Twitter。
  • 飞桨（PaddlePaddle）：百度。
  • MindSpore：华为。用于端边云场景
  • Chainer：日本Preferred Networks。非常适合进行一些复杂的决策或推理任务。
  • MXNet：亚马逊、华盛顿大学和卡内基·梅隆大学。MXNet支持混合使用符号和命令式编程。

模块化神经网络框架：建立在基础框架之上的高度模块化的神经网络库。

  • 基于TensorFlow和Theano的Keras（Keras已经被集成到TensorFlow 2.0版本中）
  • 基于Theano的Lasagne
  • 面向图结构数据的DGL

1.8 总结与深入阅读

传统机器学习中，开发一个实际的机器学习系统，除了模型和算法，人们往往花费大量的精力去做特征工程。如何自动学习有效的数据表示成为了机器学习中的关键问题。早期的表示学习方法，如特征抽取和特征选择，都是人工引入特征进行学习，这种表示学习并不是端到端的学习，且效果不能保证有效。为了解决这一问题，提出了深度学习。

深度学习里要解决的问题是贡献度分配问题，为了解决这一问题，发现神经网络模型恰好有效。

目前，深度学习主要以神经网络模型为基础，研究如何设计模型结构，如何有效地学习模型的参数，如何优化模型性能以及在不同任务上的应用等。

2 机器学习概述

ch01的1.2节总结了一些机器学习的知识。

机器学习研究的是如何从数据中训练和给定新的数据如何预测的问题。

机器学习早期叫模式识别（Pattern Recognition， PR），做一些识别的任务，后来机器学习的应用面广了，模式识别只是其中的一种应用，于是这一类问题及其解决办法就改叫机器学习了。

2.1 基本概念

  • 特征/属性（Feature/Attribute）：一个项目（item）的属性
  • 标签（Label）：对一个项目的评价，如好or坏、良性or恶性（离散值），评分、程度（连续值）。
  • 样本/示例（Sample/Instance）：一个标记好特征以及标签的项目
  • 数据集（Data Set）：一组样本的集合。数据集一般分为训练集和测试集两部分。
  • 训练集（Training Set）：一组用来训练模型的样本（即训练样本（Training Sample））
  • 测试集（Test Set）：一组用来检验模型好坏的样本（即测试样本（Test Sample））
  • 表示
    • 特征向量（Feature Vector）：一个项目的所有特征所构成的D维向量 x=[x1,x2,...,xD]Tx = [x_1, x_2, ..., x_D]^Tx=[x1​,x2​,...,xD​]T ，其中D是一个项目的特征数，每一维 xix_ixi​ 表示一个特征。
    • 标签：常用标量 yyy 表示。
    • 训练集：D={(x(1),y(1)),x(2),y(2)),...,x(N),y(N))}D = \{(x^{(1)},y^{(1)}), x^{(2)},y^{(2)}),...,x^{(N)},y^{(N)})\}D={(x(1),y(1)),x(2),y(2)),...,x(N),y(N))}
    • 训练集样本数：NNN，每个样本都是独立同分布的（Identically and Independently Distributed, IID）
    • 测试集：D′D'D′，每个样本也是独立同分布的

Q：我们想用机器学习干嘛？

A：预测/推断。给一个样本（特征向量），能预测出我想预测的东西（标签）。我们希望找到特征向量 xxx 和标签 yyy 之间的真实映射关系，这种关系我们想用函数来表示，但没有哪种函数能完全符合现实情况，只能找到一个相对最优函数 f∗(x)f^*(x)f∗(x) 来近似表达这种映射关系。如果找到此函数，那给定一个样本 xxx ，就可以预测其标签值 y^=f∗(x)\hat y = f^*(x)y^​=f∗(x) （连续），或者标签的条件概率 p^(y∣x)=fy∗(x)\hat p(y|x) = f_y^*(x)p^​(y∣x)=fy∗​(x)（离散）。这里的函数f∗(x)f^*(x)f∗(x) 要从一个函数集合F={f1(x),f2(x),...}\mathcal{F} = \{f_1(x), f_2(x), ...\}F={f1​(x),f2​(x),...} 中找，至于F\mathcal{F}F 是什么，就把它就想象成所有可能的函数组成的一个大集合吧。

Q：如何找到这个函数？（这就是机器学习的关键！）

A：学习算法（Learning Algorithm）A\mathcal{A}A，算法执行的过程叫做学习（Learning）/训练（Training）

这样，我们通过学习算法得到了真实映射关系的函数，通过给定预测样本，可以得到预测值。

2.2.1 模型

Q：我们想做一个机器学习任务，首先要看什么？

A：输入空间 𝒳 和输出空间 𝒴 。有了手头上的输入（输入空间默认为样本的特征空间），你想要什么样的输出？什么样的输出决定了我们要做什么样的机器学习任务，可以说输出空间和学习任务类型是相对应的。比如在二分类问题中 𝒴 = {+1,−1}，在 𝐶 分类问题中 𝒴 = {1,2,⋯,𝐶}，而在回归问题中 𝒴 = ℝ 。

Q：确定了𝒴之后，就要想怎样从𝒳得到（映射到）𝒴，即映射关系是什么？

A：如2.1节所述，从假设空间 ℱ（Hypothesis Space）里选一个最佳的假设（Hypothesis）𝑓∗ ∈ ℱ，这里的𝑓∗就是模型（Model），就是我们要找的映射关系。

Q：模型的公式化表示？

A：假设空间ℱ 通常为一个参数化的函数族 ℱ = {𝑓(𝒙;𝜃)|𝜃 ∈ ℝ𝐷}，其中𝑓(𝒙;𝜃)是参数为𝜃的函数，也称为模型（Model），𝐷为参数的数量。

因为常见的假设空间分为线性和非线性两种，所以模型也就分为线性模型和非线性模型。

  • 线性模型：线性模型的假设空间为一个参数化的线性函数族，即 𝑓(𝒙;𝜃) = 𝒘T 𝒙 + 𝑏，其中参数𝜃包含了权重向量𝒘和偏置𝑏。对于分类问题，一般为广义线性函数𝑦 = 𝑔(𝑓(𝒙; 𝒘))（第3章会讲）

  • 非线性模型：广义的非线性模型可以写为多个非线性基函数𝜙(𝒙)的线性组合𝑓(𝒙;𝜃) = 𝒘T 𝜙(𝒙) + 𝑏，其中𝜙(𝒙) = [𝜙1(𝒙),𝜙2(𝒙),⋯,𝜙𝐾(𝒙)]T 为𝐾 个非线性基函数组成的向量，参数𝜃包含了权重向量𝒘和偏置𝑏。如果𝜙(𝒙)本身为可学习的基函数，那𝑓(𝒙;𝜃)就等价于神经网络模型。相当于多层的（线性模型外加非线性函数）。

现在有输入空间和输出空间，根据想要的输出空间选定一种模型，接下来也就是让模型拟合训练集上的数据（学习），拟合好了模型学到功能了就可以使用了就可以预测新样本了。那模型如何学习？又怎样判断模型学习好了？这就是接下来要探讨的机器学习的三个基本要素之一——学习准则的问题。

相同的模型也可以有不同的学习算法。比如线性分类模型有感知器、Logistic 回归和支持向量机，它们之间的差异在于使用了不同的学习准则和优化算法。

2.2.2 学习准则

回顾一下，2.1节提到过训练集 DDD 中的 NNN 个样本都是IID呈独立同分布的，这是个假设，基于这个假设，我们想象所有训练集的样本都是从一个真实分布中抽样得到的，这个真实分布就是真实映射函数𝑦 = 𝑔(𝒙)或是真实条件概率分布𝑝𝑟(𝑦|𝒙)。而这种真实很难找到，我们只能找到近似的假设函数，也就是模型𝑓(𝒙;𝜃)。

既然假设式和真实式都有了，那自然想到衡量模型的好坏就是衡量模型与真实式有多相似，更具体点说是一个好的模型𝑓(𝒙,𝜃∗) 应该在所有(𝒙,𝑦) 的可能取值上都与真实映射函数𝑦 = 𝑔(𝒙)一致或与真实条件概率分布𝑝𝑟(𝑦|𝒙)一致。即|𝑓(𝒙,𝜃∗) − 𝑦| < 𝜖, ∀(𝒙,𝑦) ∈ 𝒳 × 𝒴，|𝑓𝑦(𝒙,𝜃∗) − 𝑝𝑟(𝑦|𝒙)| < 𝜖, ∀(𝒙,𝑦) ∈ 𝒳 × 𝒴（这里计算两个分布的相似性更好的方式是使用KL散度或交叉熵），其中𝜖是一个很小的正数，𝑓𝑦(𝒙,𝜃∗)为模型预测的条件概率分布中𝑦对应的概率。

过拟合（Overfitting）与欠拟合（Underfitting）

当我们给定的数据集足够多，经验风险就无限逼近期望风险，然而这不符合实际。实际我们只能获得有限样本且样本中包含一定噪声，不能很好反映全部数据的真实分布。使用经验风险最小化原则很容易导致模型在训练集上错误率很低，但在未知数据上错误率很高，这就是过拟合（Overfitting）。与过拟合对立的概念是欠拟合（Underfitting），即模型不能很好地拟合训练数据，在训练集上的错误率比较高。

过拟合问题往往是由于训练数据少和噪声以及模型能力强等原因造成的，欠拟合一般是由于模型能力不足造成的。

结构风险最小化（Structure Risk Minimization，SRM）准则

为了解决过拟合问题，就要在经验风险最小化的基础上再引入参数的**正则化（Regularization）**来限制模型的能力，使其不要拟合的那么完美，这种准则叫结构风险最小化（Structure Risk Minimization，SRM）准则。

其中‖𝜃‖是ℓ2 范数的正则化项，用来减少参数空间，避免过拟合；𝜆用来控制正则化的强度。

关于正则的具体介绍后续再讲。

2.2.3 优化算法

在确定了训练集𝒟、假设空间ℱ 以及学习准则后，如何找到最优的模型𝑓(𝒙,𝜃∗)就成了一个最优化（Optimization）问题。机器学习的训练过程其实就是最优化问题的求解过程。优化是指用优化算法去解决问题。

优化指的是模型参数的优化，即优化模型𝑓(𝒙,𝜃)中的𝜃，要与超参数做区分，超参数是用来定义模型结构或优化策略的，超参数的优化一般靠人为按经验设定。

机器学习算法想要把优化目标构造成凸函数，因为凸函数有全局最优解，凸优化也有很多成熟的优化方法。像神经网络这种模型的优化目标是非凸的，只能找局部最优解。

提前停止（Early Stop）策略

为了防止过拟合，除了2.2.2节提到的添加正则化项的方法，还可以使用提前停止（Early Stop）的策略——使用验证集进行模型选择，如果模型在验证集上的错误率不再下降就停止迭代。这样做是为了保证不在训练集上过拟合。
